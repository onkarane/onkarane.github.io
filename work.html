Customer Segmentation
Summary

Segmented Customers into 3 Clusters:

  • Bogo Group: Comprised mostly of female customers who received a lot of Buy One Get One offers. This group had the highest income, highest expenditure and became members between 2015 to 2017.
  • Passive Group: Comprised mostly of male customers who received highest number of informational offers. This group had good view rate for the offers but they hardly ever opted in and completed transactions using these offers. They also had lowest income of any group and only recently became members.
  • Discount Group: Comprised mostly of customers who had gender marked as other followed by females, and received highest number of discount offers. This group completed highest number of transactions and finally, these customers have been the part of starbucks consistently since 2013.
Technologies Used
  • Primary Programming Language: Python
  • Data Manipulation: Pandas
  • Data Visualization:Matplotlib & Seaborn
  • Scaling: Scikit-Learn Standard Scaler
  • Dimension Reduction: Scikit-Learn Principle Component Analysis (PCA)
  • Clustering: Scikit-Learn K-Means and Silhouette Score as metric

Soccer Analytics Dashboard
Summary

Development highlights include:

  • Scrapping: Used FireFox headless browser and Selenium to scrape premier league website. The elements to be extracted were identified using their tags, xpath, class names, appropriate sleep was included in the script to make sure all the required details are loaded and captured properly.
  • Data Cleaning: Filtered extra data extracted such as converting goal minutes into integer values by removing ' behind the number, converted extra time i.e 90 + 3 into proper format i.e 93, stripped unnecessary information and retrieved player name.
  • Data Storing: Categorized the scrapped data into players.csv for player information, match.csv for match information and stats.csv for match stats. In addition, the script is designed to generate unique ids for each player, match and stats previous and future both which can act as primary keys and foreign keys.
  • Data Update: The script runs on 8pm every Wednesday, Saturday, Sunday using Crontab so that the data is always updated and so is the visualization.
  • Data Visualization: The Dashboard is created in Tableau and displays game stats for each team Chelsea has faced for the year 2020-21. It also shows Chelsea's top players and win,loss,tie count. Their win count has apparently improved since hiring new manager in 2021.
Technologies Used
  • Primary Programming Language: Python
  • Scrapping: Selenium
  • Data Manipulation: Pandas
  • Visualization: Tableau
  • Scheduling: Crontab

RFM Analysis & Clustering
Summary

Development highlights include:

  • Data Cleaning: A significant amount of time was put into identifying duplicate transactions, null values related to customers such as customer id, irrelevant transaction not related to customer such as postage, carrier fee incurred by the company.
  • RFM: Each customer was put in an RFM Segment such as 444 and from that an associated RFM score was calculated. Customers having score of 12 and 9 and greater than 9 were top customers, while those having score of less than 9 and equal to 6 were the customers with potential finally customers with score less than 6 needed attention.
  • Data Preprocessing: Fixed right skewness of the data by applying log transformation. Before, log transformation was applied, fixed negative and 0 values. Finally scaled the values to standardize the mean and variance.
  • Clustering: Using K-Means Clustering segmented customers into 5 distinct clusters. For cluster 0 and 4 recency was most important factor, for cluster 1 and 3 frequency followed by monetary value were important, while for cluster 2 frequency was important.
Technologies Used
  • Primary Programming Language: Python
  • Data Manipulation: Pandas
  • Scaling: Scikit-Learn Standard Scaler
  • Dimension Reduction: Scikit-Learn Principle Component Analysis (PCA)
  • Clustering: Scikit-Learn K-Means and Silhouette Score as metric
  • Data Visualization:Matplotlib & Seaborn

Chat Bot
Summary

Development highlights include:

  • Data: Most of the data required to train the chat bot model was generated manually by identifying common questions, and the details the bot was supposed to display and stored in intents.json file.
  • Design & Training: The python script checks if the intents.json file was modified after the model was trained if so the model will be retrained or the script will simply retrieve saved model. The model is trained using Deep Neural Networks (DNN) from the Tflearn library that uses tensorflow. Additional measures have been included to make sure the reply given by the model has at least a confidence level of 70%
  • Web-App: The model is deployed to heroku cloud using the flask web-framework.
  • Website Integration: The bot is integrated into the website by using JQuery 'get' requests. The chat bot interface can thus be asynchronously updated. Additional, jquery is embedded in the site to wake the server from idle state once the user access the website.
Technologies Used
  • Primary Programming Language: Python
  • Modeling: Tensorflow 2
  • Web-Framework & Cloud: Flask & Heroku
  • Website-Integration: JQuery Get requests
